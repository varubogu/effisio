# オンボーディングガイド

新しくEffisioプロジェクトに参加されたメンバーへようこそ！

## 最初の1週間

### Day 1: アクセス権限とセットアップ

1. **アクセス権限の取得**
   - [ ] GitHubリポジトリへのアクセス
   - [ ] Slackチャンネルへの参加
   - [ ] プロジェクト管理ツールへのアクセス

2. **開発環境のセットアップ**
   - [ ] [開発環境セットアップガイド](./DEVELOPMENT_SETUP.md) に従って環境構築
   - [ ] docker-compose up で動作確認
   - [ ] ローカルでアプリケーションが起動することを確認

3. **ドキュメント確認**
   - [ ] README を読む
   - [ ] プロジェクト概要（plans/01_project_overview.md）を読む
   - [ ] 技術スタック（plans/02_tech_stack.md）を確認

### Day 2-3: コードベースの理解

1. **ドキュメント学習**
   - [ ] [Git運用ルール](./GIT_WORKFLOW.md)
   - [ ] [コーディング規約（Go）](./CODING_GUIDELINES_GO.md)
   - [ ] [コーディング規約（TypeScript）](./CODING_GUIDELINES_TYPESCRIPT.md)

2. **コードベースの探索**
   - [ ] ディレクトリ構造を確認
   - [ ] 主要なコンポーネントを読む
   - [ ] テストコードを読む

### Day 4-5: 最初のタスク

1. **簡単なタスクに取り組む**
   - [ ] ドキュメントの誤字修正
   - [ ] 簡単なバグ修正
   - [ ] テストの追加

2. **プロセスの実践**
   - [ ] ブランチ作成
   - [ ] コミット・プッシュ
   - [ ] PR作成
   - [ ] コードレビューを受ける

## チェックリスト

### 開発環境
- [ ] Go 1.21+ インストール済み
- [ ] Node.js 20+ インストール済み
- [ ] Docker Desktop 起動済み
- [ ] VSCode（または好みのエディタ）設定済み
- [ ] Git設定完了

### プロジェクト理解
- [ ] プロジェクトの目的を理解
- [ ] 主要な機能を把握
- [ ] 技術スタックを理解
- [ ] ディレクトリ構造を把握

### プロセス理解
- [ ] Git Flow を理解
- [ ] コミットメッセージ規約を理解
- [ ] PR作成方法を理解
- [ ] コードレビュープロセスを理解
