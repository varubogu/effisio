# リリースチェックリスト

## リリース準備

### コード品質
- [ ] すべてのテストが通っている
- [ ] リントエラーがない
- [ ] コードレビューが完了している
- [ ] すべてのPRがマージされている

### ドキュメント
- [ ] CHANGELOG が更新されている
- [ ] バージョン番号が更新されている
- [ ] API ドキュメント（Swagger）が最新
- [ ] README が最新

### テスト
- [ ] ユニットテスト: ✅
- [ ] 統合テスト: ✅
- [ ] E2Eテスト: ✅
- [ ] 手動テスト: ✅
- [ ] ステージング環境での動作確認: ✅

### セキュリティ
- [ ] セキュリティチェックリスト完了
- [ ] 脆弱性スキャン実施
- [ ] 機密情報がコミットされていない

### パフォーマンス
- [ ] 負荷テスト実施
- [ ] レスポンスタイムが基準内
- [ ] データベースインデックス最適化

## リリース手順

1. [ ] release ブランチ作成
2. [ ] バージョン番号更新
3. [ ] CHANGELOG 更新
4. [ ] PR 作成・レビュー
5. [ ] main ブランチへマージ
6. [ ] Git タグ作成・プッシュ
7. [ ] デプロイ実行
8. [ ] 動作確認

## リリース後

- [ ] 本番環境の動作確認
- [ ] ログ確認
- [ ] エラー監視
- [ ] パフォーマンス監視
- [ ] チームへの通知

## ロールバックプラン

問題が発生した場合:
1. 前のバージョンにロールバック
2. 問題を調査
3. 修正版をデプロイ
